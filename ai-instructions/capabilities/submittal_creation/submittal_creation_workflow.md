# Workflow: Create a Submittal Cover Page and Final Package

## 1. Intake

When the user asks to create a submittal:

- Confirm a supporting document is available.
- If not, ask the user to upload it.
- Determine whether the request is for a new submission or a resubmittal.

Do not finalize a package without reviewing the supporting document.

---

## 2. Review the Supporting Document

Read the full contents of the uploaded document. Do not rely only on the filename or the user's description.

Extract and record every item that maps to a cover page field:

| Cover Page Field | What to Look For in the Document |
|---|---|
| Document type | Format of the document (cut sheet, fabrication drawing, calc, etc.) |
| Item Description | Product name, model, material, system, or work scope |
| Manufacturer / Supplier | Company name, logo, letterhead, title block |
| Spec./PRDC Reference | Any section number or specification citation in the document |
| Drawing Number | Drawing number, revision, and title from title block |
| Location Where Used | Station name, platform, room, line, or area |
| Review Purpose | Stated intent: approval, review, information, etc. |
| Buy America | Any Buy America or Buy American compliance statement |
| Deviations | Substitutions, dimensional differences, alternate finishes, missing items |

Also note:

- whether the document is complete or missing pages;
- whether certifications, seals, or test data are required and present.

---

## 3. Classify the Submission

Use the document contents to classify the package.

### Product Data
Manufacturer cut sheets, data sheets, literature, installation instructions, compliance information, or technical specifications.

### Shop Drawing
Fabrication drawings, layouts, dimensions, details, coordinated drawings, or installation drawings.

### RFI
A request for clarification, interpretation, or direction concerning the contract documents.

### Transmittal
A formal transmission of documents or information where a separate approval submittal is not required.

### Letter to Owner
Formal correspondence directed to the owner, designer, construction manager, agency, or another project stakeholder.

### Other
Calculations, test reports, certifications, samples, closeout documents, or informational packages.

If uncertain, compare the file with similar existing project documents.

---

## 4. Determine the Specification Section

All submittal specification sections follow **CSI MasterFormat** — the universal construction specification numbering standard organized by Division (two digits) and Section (six digits), e.g., `06 60 00`.

Use this resolution order:

1. **Section stated in the supporting document** — check title blocks, headers, cover sheets, and notes.
2. **Section stated by the user.**
3. **Project submittal log** — check for an existing entry for this product or work type.
4. **Existing folder name** — the destination folder may already carry the correct section number.
5. **Similar prior submittals** — find the closest matching product or work type already numbered in the project.
6. **CSI MasterFormat lookup** — identify the product type, material, or work scope and match it to the correct Division and Section:
   - Division 03 – Concrete
   - Division 04 – Masonry
   - Division 05 – Metals
   - Division 06 – Wood, Plastics, and Composites
   - Division 07 – Thermal and Moisture Protection
   - Division 08 – Openings
   - Division 09 – Finishes
   - Division 10 – Specialties
   - Division 22 – Plumbing
   - Division 23 – HVAC
   - Division 26 – Electrical
   - Division 31–35 – Civil, Site, Transportation
   *(and all other standard divisions)*
7. **Internet search** — if the section cannot be confirmed from project files or CSI knowledge alone, search the web using the product name, manufacturer, material type, or system description to identify the standard CSI MasterFormat section. Cite the source.

Do not assign a section number when evidence is insufficient. State which source was used and flag it as confirmed or inferred.

For the reference example, the section is:

`06 60 00 – Edge Strip Rubbing Board`

This was confirmed from the document title block and CSI Division 06 (Wood, Plastics, and Composites).

---

## 5. Select the Example Cover Page

Search for the most suitable project example using this order:

1. same project, same type, same spec section;
2. same project, same type, similar section;
3. same project, current cover-page template;
4. approved project template.

Use the newest reliable format.

Preserve:

- logos;
- title block;
- page size;
- fonts;
- tables;
- checkboxes;
- signature/review fields;
- note wording;
- footer/header;
- margins;
- overall visual layout.

---

## 6. Determine the Destination Folder

Use the applicable spec section and the project's existing folder structure.

The folder may be controlled by:

- spec section;
- discipline;
- document type;
- station;
- submission status;
- revision status.

Do not create a new folder structure unless the user authorizes it.

---

## 7. Assign the Next Number

Review all files in the destination folder.

Identify:

- highest issued base number;
- skipped numbers;
- duplicate numbers;
- revisions;
- drafts;
- voided or archived packages.

For a new submission, use the next unused sequential base number.

For example:

- existing: `GEN-001`, `GEN-002`, `GEN-003`
- next: `GEN-004`

For a resubmittal:

- keep the base number;
- increment the revision only.

Example:

- original: `A37806_06 60 00_GEN-001R00`
- first resubmittal: `A37806_06 60 00_GEN-001R01`

Never reuse an issued number.

---

## 8. Complete the Cover Page

Populate all applicable fields using the rules below.

### Review Purpose

- **For Designer Approval** — check this box by default for all submittals except workplans.
- **For NYCT/MTA Review & Comment**, **For NYCT/MTA Approval**, **For NYCT/MTA Information Only**, **For Designer Information Only** — check only when the document type or user instruction specifically requires it.
- **Workplans** — do not pre-select a review purpose; ask the user which box to check.

### Buy America

- **Always check the Buy America Compliant box.** This is the project standard for all submittals on A37806.

### PM and Quality Manager Signatures

- Copy the PM and QM signature lines directly from `example_submittal.pdf` (Ravi Jain / Nabeel Anjum with existing signatures).
- Do not leave these fields blank and do not alter the signatures.
- These fields are standing project reviewer fields and carry over to every cover page.

### Remaining Fields

Populate from the supporting document and project data:

- Date
- Contract Number: `A37806`
- Contract Title: `Design-Build Services for Accessibility Upgrades - Package 6`
- Item Description *(see Step 9)*
- Submittal Number *(assigned in Step 7)*
- Revision
- Spec./PRDC Reference *(confirmed in Step 4)*
- Location Where Used
- Drawing Number
- Number of Pages Including Cover Sheet
- Notes

### Standard note language

`This submittal has been reviewed by the Project Manager and Quality Manager to certify that it meets the contract requirements.`

---

## 9. Write a Clear Item Description

Use a specific item description.

Good:
- `Rubbing Board for Burnside Ave and Middletown Road`
- `Platform Window Product Data`
- `Northbound Stair Enclosure Framing Shop Drawings`

Avoid:
- `Product Data`
- `Shop Drawing`
- `Material`
- `Submittal`

---

## 10. Check for Deviations and Missing Information

Review for:

- manufacturer substitutions;
- thickness or dimension differences;
- alternate colors or finishes;
- missing certifications;
- missing testing;
- missing seals;
- missing drawings;
- incomplete locations;
- inconsistent specification references;
- incorrect station or division;
- Buy America uncertainty.

If no deviation is identified, use cautious wording such as:

`No known deviations from the Contract Documents based on the submitted information.`

Do not state that there are no deviations unless reasonably supported.

---

## 11. Assemble the Final PDF

The final order should be:

1. cover page;
2. contractor review sheet, if separate;
3. table of contents, if needed;
4. supporting product data or drawings;
5. calculations;
6. certifications;
7. test reports;
8. other attachments.

The cover page must be page 1.

Preserve:

- searchable text;
- bookmarks;
- drawing quality;
- page orientation;
- original technical content.

Update the page count to include the cover page.

---

## 12. Rename the Final File

Follow the project's existing naming convention.

Based on the reference example:

`[Contract]_[Spec Section]_[Category]-[Sequence][Revision] - [Status] - [Item Description].pdf`

Example:

`A37806_06 60 00_GEN-001R00 - ORIG - Rubbing Board for Burnside Ave and Middletown Road.pdf`

Where:

- `A37806` = contract number;
- `06 60 00` = specification section;
- `GEN` = category code;
- `001` = sequence number;
- `R00` = revision;
- `ORIG` = original submission;
- final text = item description.

For resubmittals, follow the project's existing status wording, such as `RESUB`, only when confirmed by similar files.

The filename, cover page number, revision, title, and folder must agree.

---

## 13. Validate Before Completion

Check all of the following:

- correct project;
- correct document type;
- correct spec section;
- correct destination folder;
- next number is unused;
- revision is correct;
- title is specific;
- cover page matches current template;
- all fields agree with the supporting document;
- cover page is first;
- total page count is correct;
- final PDF opens;
- pages are legible and correctly oriented;
- filename follows project convention;
- no existing issued file was overwritten;
- no unsupported information was invented.

---

## 14. Completion Message

Report:

- classification;
- spec section;
- assigned submittal number;
- revision;
- template used;
- final filename;
- destination folder;
- included files;
- assumptions;
- missing information;
- deviations or concerns.

Example:

`Created Product Data Submittal A37806_06 60 00_GEN-004, Revision 0, using the current A37806 cover-page format. The cover page was placed as page 1 and combined with the supporting product data. Proposed filename: A37806_06 60 00_GEN-004R00 - ORIG - [Item Description].pdf. Proposed folder: Submittals/06 60 00/Pending. Buy America compliance and reviewer names remain to be confirmed.`

---

## 15. Required Guardrails

- Never invent critical project data.
- Never duplicate a number.
- Never overwrite an issued file.
- Never call a draft "approved" or "issued."
- Never hide known deviations.
- Never assign a new base number to a revision unless the project convention requires it.
- Ask only for information that cannot be reliably determined from the project files.
